# Web Attacks模块说明


本模块讲解三类常见的 Web 漏洞：HTTP 方法篡改（HTTP Verb Tampering）、不安全直接对象引用（IDOR） 和 XML 外部实体注入（XXE），每类漏洞都可能对企业系统造成严重影响。我们将通过多种方式，讲解如何对这三类漏洞进行识别、利用与防护。
模块总结
随着 Web 应用的普及度不断提升，针对 Web 应用的攻击数量与类型也在持续增长。许多最常见的 Web 攻击已在其他 Web 安全模块中讲解过，本模块将介绍另外几种重要的 Web 漏洞利用技术。
模块首先讲解HTTP 方法篡改（HTTP Verb Tampering）漏洞，这类漏洞允许攻击者篡改 HTTP 请求，从而访问受限制的页面，进而获取敏感资源。
第二个主题是不安全直接对象引用（Insecure Direct Object References, IDOR）漏洞。IDOR 是 Web 应用中最常见的漏洞之一，由于访问控制机制不完善，攻击者可轻易查看并泄露其他用户的隐私数据。
最后，模块将讲解XML 外部实体注入（XML External Entity, XXE）漏洞。该类漏洞近年已跻身十大最危险 Web 漏洞之列，攻击者可利用 XXE 读取服务器本地文件、获取远程代码执行（RCE）权限，甚至造成拒绝服务（Denial of Service, DoS）。

# Attacking Common Applications模块介绍

渗透测试人员可能会遇到各种各样的应用程序，例如内容管理系统、自定义 Web 应用程序、开发人员和系统管理员使用的内部门户等等。在许多不同的环境中发现相同的应用程序是很常见的。虽然某个应用程序在一个环境中可能不存在漏洞，但在另一个环境中可能存在配置错误或未打补丁的情况。作为评估人员，必须熟练掌握本模块中讨论的常见应用程序的枚举和攻击方法。这些知识将有助于在评估过程中遇到其他类型的应用程序时进行应对。

本模块涵盖了在内部和外部渗透测试中经常遇到的各种应用程序的发现、足迹分析、枚举和攻击所需的各种技术。

本模块将涵盖以下内容：

* 应用程序发现与枚举
* 枚举并攻击常见的 CMS 系统，例如 WordPress、Drupal 和 Joomla
* 枚举并攻击 Tomcat 和 Jenkins
* 枚举和攻击基础设施工具，例如 Splunk 和 PRTG 网络监控器
* 枚举并攻击客户服务管理和配置管理工具，例如 osTicket 和 GitLab。
* 其他常见应用
* 应用强化核心概念
