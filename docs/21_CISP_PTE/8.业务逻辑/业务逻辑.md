# 第一题

用户已经通过弱口令得知了用户名admin密码admin123 ，但是服务端却限制IP地址，该如何伪造地址。

部分源码如下：

```php
header('Content-Type: text/html; charset=utf-8');

$cip = $_SERVER['HTTP_CLIENT_IP'];

if ($cip != "1.2.3.4") {

echo "非法IP地址，只有1.2.3.4才能访问！";

exit();

}



$username = $_POST['username'];

$password = $_POST['password'];
```

## write up

当使用弱口令登录时，提示：

![image-20260402215115771](./image/%E4%B8%9A%E5%8A%A1%E9%80%BB%E8%BE%91/image-20260402215115771.png)

本题主要考查的是ip绕过，常见在http 请求头中填写以下内容：

1. x-forwarded-for: 127.0.0.1
2. client-ip: 127.0.0.1

本题考察第二个，我们只要在请求头中增加 `client-ip: 1.2.3.4` 即可绕过ip限制

![image-20260402215603272](./image/%E4%B8%9A%E5%8A%A1%E9%80%BB%E8%BE%91/image-20260402215603272.png)

# 第三题

题目：Admin Panel 的隐秘后门

描述：你能发现管理员面板的隐藏访问方式吗？

目标：删除用户tom 获取 flag

## write up

提供了一个登录界面，首先尝试弱口令登录

![image-20260402220326222](./image/%E4%B8%9A%E5%8A%A1%E9%80%BB%E8%BE%91/image-20260402220326222.png)

找不到弱口令

尝试扫目录，除了login 和admin 没有扫到其他的目录

虽然管理员面板提示需要管理员或者本地访问，但是尝试修改添加了以下几个，都不行

```http
Referer: http://127.0.0.1:34720/
x-forwarded-for: 127.0.0.1
client-ip: 127.0.0.1
real-ip: 127.0.0.1
```

没辙了，看答案

本题考察 http的请求方法中的 trace 方法，是服务器原样回显请求包的调试方法，当前题目中请求为这样的

![image-20260402224136014](./image/%E4%B8%9A%E5%8A%A1%E9%80%BB%E8%BE%91/image-20260402224136014.png)

可以看到除了我们请求的原始请求头信息外，还包含了两个额外的信息，可以推测 他可能是反向代理添加上去的，如果我们伪造

==X-Custom-IP-Authorization== 字段为本地ip地址，是不是就能绕过呢？

![image-20260402224451085](./image/%E4%B8%9A%E5%8A%A1%E9%80%BB%E8%BE%91/image-20260402224451085.png)

可以看到，确实成功了
