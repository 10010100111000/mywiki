# 一、IAAA模型

IAAA 模型是一种简化的用户及其行为验证方式的思路。每个步骤都至关重要，且不可跳过任何步骤。也就是说，如果前面的步骤没有执行，则无法执行后续步骤。这四个步骤分别是：

- **Identity** - **身份**代表个人或服务的唯一帐户（例如，用户 ID/电子邮件）。
- **Authentication** - **身份验证**证明身份（密码、一次性密码、通行密钥）。
- **Authorisation** －**授权**该身份被允许执行的操作。
- **Accountability** －**问责制** 记录并通报谁在何时何地做了什么。

本会议室讨论的 OWASP Top 10:2025 中的三个类别都与 IAAA（ 身份、授权和访问控制）的实现缺陷有关。这些缺陷可能造成极其严重的后果，因为它们可能使攻击者能够访问其他用户的数据或获得超出其应有权限的权限。

# 二、A01：Broken Access Control

**访问控制失效**是指服务器未能正确执行每次请求的**访问权限**控制。这种情况的常见表现是：IDOR(不安全的直接对象引用)：如果更改 ID（例如 `?id=7 → ?id=6` ）可以让您查看或编辑其他人的数据，则访问控制已失效。

访问控制失效漏洞是指访问控制机制未能有效限制用户对资源或数据的访问。以下是一些常见的访问控制失效漏洞及其示例：

1. **横向权限提升**是指攻击者能够访问其他具有相同访问权限级别的用户的资源或数据。例如，用户可能通过更改 URL 中的用户 ID 来访问其他用户的帐户。

2. **垂直权限提升**是指攻击者能够访问属于更高权限级别用户的资源或数据。例如，普通用户可以通过操纵隐藏的表单字段或 URL 参数来访问管理功能。

3. **访问控制检查不足**是指访问控制检查执行不正确或不一致，从而允许攻击者绕过这些检查。例如，应用程序可能允许用户在未验证其权限的情况下查看敏感数据。

4. **不安全的直接对象引用**是指 当攻击者能够利用应用程序访问控制机制中的漏洞，从而访问某项资源或数据时，就会发生**不安全的直接对象引用**。例如，应用程序可能会对敏感数据使用可预测或容易被猜测的标识符，从而使攻击者更容易访问这些数据。

   

# 三、A07: Authentication Failures

当应用程序无法可靠地验证或绑定用户身份时，就会发生身份验证失败。常见问题包括：

- 用户名枚举
- 弱密码/易猜密码（无锁定/速率限制）
- 登录/注册流程中的逻辑缺陷
- 不安全的会话或 Cookie 处理
- 如果存在上述任何一种情况，攻击者通常可以以其他人的身份登录，或者将会话绑定到错误的帐户。

# 四、A09: Logging & Alerting Failures

当应用程序不记录或发出安全相关事件的警报时，防御者就无法检测或调查攻击。良好的日志记录是**问责制的**基础（能够证明谁在何时何地做了什么）。在实践中，日志记录失败的表现形式包括：缺少身份验证事件、错误日志模糊不清、没有针对暴力破解或权限变更的警报、日志保留时间过短，或者日志存储在攻击者可以篡改的位置。
