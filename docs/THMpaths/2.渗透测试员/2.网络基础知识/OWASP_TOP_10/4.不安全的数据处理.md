# A04.Cryptographic failiures

加密机制失效 指的是当敏感数据由于缺乏加密、实施错误或安全措施不足而未得到充分保护时，就会发生加密故障。这包括未对密码进行哈希处理而直接存储密码、使用过时或弱算法（例如 MD5 、SHA1 或 DES ）、泄露加密密钥或在传输过程中未能确保数据安全。

# A05.Injection

注入攻击一直是 OWASP 十大安全漏洞之一，这并不令人意外。注入攻击仍然是 Web 漏洞利用的经典案例。

当应用程序获取用户输入并错误处理时，就会发生注入攻击。应用程序没有安全地处理输入，而是将其直接传递给可以执行命令或查询的系统，例如数据库、shell、模板引擎或 API 。

您可能对 SQL 注入攻击并不陌生，攻击者可以将一段 SQL 查询语句插入到应用程序的逻辑中，例如登录表单，然后数据库会处理这段语句。这种情况通常发生在 Web 应用程序未能对用户输入进行安全过滤，而是直接使用用户输入来构建查询语句时。例如，直接获取登录表单中的“用户名”输入并用它来查询数据库。

以下是一些你可能熟悉的经典注射示例：

- SQL Injection
- Command Injection
- AI Prompts
- Server Side Template Injection (SSTI)

不幸的是，即使到了 2025 年，这类攻击仍然具有现实意义， OWASP 十大安全漏洞列表中不仅在 2021 年出现了一次，到 2025 年更是出现了两次，就证明了这一点。注入攻击的危害性很高，应该予以相应的重视。

# A08: Software or Data Integrity Failures

软件或数据完整性故障一直是 OWASP 十大安全漏洞榜单上的老生常谈，在过去两个版本中都出现过两次。下面我们来进一步探讨一下。

软件或数据完整性故障是指应用程序依赖于它认为安全的代码、更新或数据，而未验证其真实性、完整性或来源。这包括未经验证就信任软件更新、从不受信任的来源加载脚本或配置文件、未能验证影响应用程序逻辑的数据，或者在未确认数据是否已被篡改的情况下接受二进制文件、模板或 JSON 文件等数据。
