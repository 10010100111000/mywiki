# 二、IAAA模型

身份标识 、认证、授权和问责（ IAAA ）是信息安全的四大支柱。这些要素中的每一个都对确保敏感信息和资源的机密性、完整性和可用性起着至关重要的作用。

在 IAAA 模型中， IAAA 代表身份标识 (Identification)、认证 (Authentication)、授权 (Authorization ) 和问责制 (Accountability)。IAAA 模型的四个阶段是：

1. **身份标识**  是用户向系统**宣称**（Claiming）其身份的过程。它通常通过提供一个在特定环境下唯一的**标识符**（如用户名、邮箱或工号）来实现。标识是身份访问管理的第一步，紧随其后的是**认证（Authentication）**，即系统验证该声明真实性的过程。
2. **身份验证** 是指确保用户身份与其声称的身份相符的过程。换句话说，这一步骤旨在确认用户所声称的身份。一种身份验证方法是提供正确的密码。但由于密码可能存在安全漏洞，许多其他方法，例如要求用户输入发送到其邮箱的验证码，正变得越来越流行。
3. **授权**决定了用户可以访问哪些资源。换句话说，用户将被授权根据其账户权限执行特定操作。此过程通常通过根据用户的职位或安全级别分配角色和权限来实现。通过限制用户只能访问履行职责所需的资源，可以降低未经授权访问或数据泄露的风险。
4. **问责机制**会追踪用户活动，确保用户对其行为负责。用户获得系统访问权限后，必须建立相应的机制，让每个人都对其行为负责。这一过程通过记录所有用户活动并将其集中存储在一个位置来实现。一旦发生安全事件，这些信息可用于识别问题根源并采取相应措施。

IAAA 有助于防止未经授权的访问、数据泄露和其他安全事件。通过实施这些最佳实践，组织可以保护其敏感信息和资源免受内部和外部威胁。

# 三、Identification 

身份标识 是指用户（或进程或系统）声明其特定身份的方式。让我们来看几个日常生活中的例子。假设你受邀参加一个聚会，开始与人交流，这时有人问你：“你叫什么名字？” 你很可能会告诉他们你的真实姓名。毕竟，你是来玩得开心，结交新朋友的。等等，你也可以随便取个名字，或者从你最喜欢的电影里找个名字，比如“托马斯·安德森”！无论如何，对方都不会要求你出示身份证件来验证身份；这只是一个社交活动，又不是要进入高度安全的区域。

身份识别可以通过用户名实现。用户名可以有多种形式。例如，Thomas Anderson 的用户名可以是 `tanderson` 、 `thomasa` 、 `thomas01` 、 `ta001` 甚至是 `neo` 。这完全取决于组织和平台。

任何用户独有的数字都可用于身份识别。许多网站要求注册时提供电子邮件地址，因为用户的电子邮件地址保证是唯一的；这省去了用户寻找和记住唯一用户名的麻烦。

我们不妨考虑一下计算机以外的另一个例子。假设你刚开始去一家健身房，前台问你叫什么名字。你可能会脱口而出一个你最喜欢的电影里的名字；但是，这次前台可能会要求你出示会员卡。他们为什么要这么做呢？这是为了确认你的身份，也就是身份验证。他们不想让一个陌生人随便走进健身房，就冒充付费会员。如果任何人都能进来，都自称是会员，健身房就无法盈利了。

如果没有适当的身份验证，可能会造成严重损失；例如，有人在银行贷款时使用虚假身份。在 IT 领域，如果没有身份验证，任何人只要知道你的电子邮件地址就能访问你的电子邮件。大多数系统如果没有适当的身份验证就无法正常运行；这些系统不仅限于计算机系统，还包括银行系统、酒店预订系统和航班系统等等。在接下来的任务中，我们将更详细地介绍身份验证。

# 四、Authentication

身份验证是指验证用户或系统身份的过程。让我们回到健身房的例子。健身房前台只允许已注册会员进入。在现实世界中，这是如何实现的呢？他们可以要求你出示会员卡（假设健身房为每位会员提供一张）。你应该有一张包含你的照片和相关会员信息的会员卡。前台可以通过检查你的会员卡来确认你的身份；身份验证问题就解决了（除非有人找到了伪造会员卡的方法，但这又是另一个问题了）。

身份验证和识别是任何信息系统和网络的核心组成部分。理解身份验证和识别之间的区别至关重要。
在身份识别过程中，用户（或系统或进程）在相应的设置中声明一个特定的（唯一的）身份。身份验证则是证明用户（或系统或进程）的身份。此过程通常通过以下方式之一完成：

1. Something you know 你知道的
2. Something you have 你拥有的东西
3. Something you are 你是什么

还有两种方法，但使用频率较低：

- 你所在的地方（逻辑/物理位置）
- 你做的事情（行为）

### Something You Know 

你所知道的事物指的是你了解或记住的事物。例如：

- 密码示例： `4SNoPawKkdFiCdnm` 和 `%WAdWi-;4,mxRMQB`
- 诸如 *“Judge Battle Advise Pain 9”* 和 *“Baggage Protection Dissatisfy Barrel 8”* 之类的密码短语
- 个人识别码 (PIN)，例如 `25063` 和 `6285`

大多数手机在闲置几分钟后会自动锁定。根据初始设置，用户可以通过输入正确的 PIN 码、密码或图案来解锁。虽然图案是画出来的，但它与 PIN 码并无本质区别，都是需要记住的。

### Something You Have

你所拥有的东西指的是你拥有的物品，通常是实物。它可以是手机，也可以是安全钥匙。

例如，当您注册某些即时通讯应用时，系统会要求您提供电话号码，通常是手机号码。这个电话号码就是您在该应用上的身份标识。如何证明这确实是您的电话号码呢？一种方法是通过短信向您发送验证码，或者拨打该号码并告知您验证码。如果您在几分钟内阅读了短信或接到了该号码的来电，即可证明您*拥有*该电话号码。如果是手机号码，则只需证明您持有相应的 SIM 卡（或 eSIM 卡）即可。

硬件安全密钥体积小巧，可以挂在钥匙链或钱包里。您可以通过将安全密钥插入 USB 或 USB-C 端口，或将其靠近 NFC（近场通信）读取器来使用身份验证。硬件安全密钥的例子包括 Yubico、Titan Security Key、Nitrokey 和 Thetis 等等。

### Something You Are 

您所使用的技术指的是生物识别读取器。例如指纹读取器、面部识别、视网膜扫描仪和语音识别。您很可能体验过使用指纹识别器解锁手机。许多现代手机允许用户使用指纹进行身份验证，同时保留密码/PIN 码/图案作为备用方案，以防指纹验证失败。

面部识别技术在现代智能手机中也越来越普及。多年来，生物识别读取器和扫描仪不仅变得更加可靠，而且价格也更加亲民。这项技术既有利于需要高安全性的企业，也有利于消费者。

### 多因素身份验证(MFA)

多因素身份验证 ( MFA ) 指的是使用两种或多种上述机制（您知道的/拥有的/自身具备的）。其目的是在一种身份验证机制被攻破时提供额外的安全保障。

如果您想使用银行的自动取款机，请插入您的信用卡/借记卡并输入您的密码。此过程是最早的双因素身份验证 (2FA) 示例之一。其显而易见的优势在于，攻击者仅获得您的卡是不够的，他们还需要知道您的密码。如今市面上许多家用保险箱都内置了双因素认证（2FA），只是未必会特意宣传。这类保险箱需要用户插入钥匙并输入正确的密码才能打开。也就是说，用户必须**知道**密码并**持有**钥匙才能打开保险箱。因此，即使其他人获得了保险箱钥匙，如果没有密码，他们仍然无法打开保险箱。

在迭代过程中，**双因素身份验证 (2FA)** 需要两种身份验证机制，它属于更通用的多因素身份验证 (MFA) 范畴，后者需要两个或多个身份验证因素。这一要求可以显著提高安全性，并抵御各种攻击，例如利用弱密码的攻击。

# 五、Access control 

用户身份验证通过后，应被授予相应的访问权限。授权机制规定了已验证用户可以访问哪些资源以及可以执行哪些操作。以健身房为例，我们期望付费会员在规定的营业时间内可以使用所有训练器材。会员不能借用跑步机一个月。健身房不是图书馆！访问控制机制可以确保有效执行授权。如果健身房拥有完善的访问控制机制，就会有人发现你费力地搬运跑步机，并将其搬走。

我们来看另一个非技术性的例子：琳达预订了一家酒店房间，入住一周。根据预订信息，琳达应该有权在入住期间使用她的酒店房间以及其他公共设施。虽然她可以进入她预订的房间，但她不能进入其他房间。如何才能确保这一点呢？同样，授权是通过访问控制机制来实现的。

在酒店例子中，琳达拿到了一把钥匙，凭此钥匙可以进入她被分配的房间。因此，门禁控制是通过锁和钥匙来实现的。更先进的酒店可能会使用智能卡和电子读卡器来解锁房门。无论采用哪种方式，都存在一种机制来强制执行授权。

我们来看一个技术示例。作为销售团队的一员，Edward 需要访问所有与销售相关的必要文件，以便高效地完成工作。例如，销售团队无需访问人力资源和会计相关的文档。在这种情况下，可以通过设置公司文件和知识库的适当访问权限来实施访问控制。



# 六、Accountability

**问责制**确保用户对其在系统上执行的操作负责。换句话说，用户在验证身份并获得系统访问权限后，就必须对其行为负责。问责制的前提是具备审计能力，而**审计**能力通常需要完善的**日志记录**功能。

我们先来看一个非技术性的例子：健身房。你办了健身卡，每周去三次。现在你成了常客，前台认得你，不用你出示会员卡。就好像你一直“在线”一样！你注意到，所有“有权”进入健身房的人都遵守着一些规则。例如，如果有人对自己的进步速度不满意，就不能砸碎墙上的镜子。否则，他们的会员资格将被取消，并且需要赔偿所有损失。换句话说，每个人都要为自己的行为*负责* 。这种模式让每个人都能安全地锻炼。

我们来看一个更实际的例子，比如银行柜员。柜员可以查看和处理客户账户上的各种交易。我们如何确保不会有不法员工滥用这种授权呢？我们需要安全地记录所有交易及其相关信息。我们应该能够审计所有已完成的交易，并审查每位操作者的具体操作。如果没有这种能力，我们就无法依赖或信任这样的系统。

### Logging

问责制的一个关键方面是日志记录。日志记录是指记录系统内发生的事件的过程。这些事件包括用户操作、系统事件和错误。通过记录用户操作，组织可以维护一份记录，了解谁在何时访问了哪些信息。这份记录对于合规性、事件响应和取证调查至关重要。

通过部署完善的日志系统，组织可以追踪任何用户的操作，识别任何异常情况或未经授权的访问，并采取相应的措施。例如，如果未经授权的用户试图访问敏感数据，日志系统可以生成警报通知安全人员。

日志记录还能帮助组织检测和应对安全事件。通过分析日志数据，安全团队可以识别可疑活动模式，例如重复的登录失败尝试或异常的访问模式。这些信息随后可用于调查和应对潜在的安全威胁。

由于问责制是任何安全基础设施的关键组成部分，因此必须妥善确保日志记录正确且安全地执行。此外，根据安全要求，日志应具备防篡改功能。这是因为您不希望攻击者删除或更改日志，从而掩盖其在网络上的行为。因此，最佳实践是设置一个独立的日志服务器，其唯一任务是安全地接收和存储日志。这就是日志转发的由来。

**日志转发**是将日志数据从一个系统发送到另一个系统的过程。此过程通常会将来自多个来源的日志数据聚合到一个中心位置，以便更方便地进行分析和管理。日志转发还可以用于将日志数据发送到云服务进行存储和分析。

日志转发有诸多好处。通过集中管理日志数据，组织可以更轻松地分析和关联来自不同系统的日志事件，从而识别潜在的安全威胁。这就引出了安全信息和事件管理（ SIEM ）的概念。

### SIEM

Security Information and Event Management system（安全信息与事件管理系统）该系统用于将**日志、告警、取证数据和事件**等形式的安全信息汇集到一个集中化平台，使安全分析师能够在安全监控过程中执行近实时分析。

# 七、身份管理（IdM）访问管理（IAM）

身份管理 (IdM) 涵盖了身份识别、认证和授权所需的所有策略和技术。IdM 旨在确保授权人员能够访问其工作所需的资产和资源，同时拒绝未经授权人员的访问。IdM 要求为每个用户或设备分配一个数字身份。
身份管理 (IdM) 帮助组织保护敏感数据并确保符合法规要求。它还能帮助组织简化用户访问流程、降低身份管理成本并提升用户体验。通过实施有效的 IdM 策略，组织可以确保用户经过身份验证和授权后，能够安全地访问所需的资源。

有些资料将身份管理 (IdM) 和身份与访问管理 ( IAM ) 混用。另一些资料则认为 IdM 更侧重于与用户身份相关的安全问题，例如身份验证和权限管理。他们指出，IdM 关注的是管理用户、设备和组的属性和权限，而 IAM 则更侧重于评估这些属性和权限，并根据公司策略授予或拒绝访问权限。本文将它们视为不同的概念，尽管它们之间的界限往往比较模糊。

### 身份管理 (IdM) 

身份管理 (IdM) 是网络安全的重要组成部分，指的是管理和控制数字身份的过程。它涉及用户身份的管理、身份验证、授权和访问控制。IdM 的主要目标是确保只有授权人员才能访问特定的资源和信息。IdM 系统用于管理组织网络中的用户身份。

身份管理 (IdM) 系统使用集中式数据库来存储用户身份和访问权限。它们还提供管理和监控用户资源访问的功能。IdM 系统通常包含用户配置、身份验证和授权等功能。用户配置是指创建和管理用户帐户的过程，而身份验证和授权是指验证用户身份并授予其对特定资源的访问权限

在拥有多个需要访问控制的系统和应用程序的组织中，身份管理系统 (IdM) 至关重要。它们有助于简化用户身份管理，降低未经授权访问资源的风险。此外，IdM 系统为用户身份管理提供统一的参考点，使组织能够更轻松地管理用户访问权限。

### 访问管理（IAM）

Identity and Access Management (IAM)是一个比身份管理 (IdM) 更全面的概念。它涵盖了管理和保护数字身份及访问权限的所有流程和技术。IAM 系统包含多种功能，例如用户配置、访问控制、身份治理和合规性管理。IAM 系统确保只有授权用户才能访问特定资源和数据 ， 并且对其访问进行监控和控制。

身份与访问管理 (IAM) 系统为组织内资源的访问管理和安全提供了一套全面的解决方案。它们与多个系统和应用程序集成，提供用户身份和访问权限的集中视图。IAM 系统采用多种技术来管理访问权限，包括基于角色的访问控制、多因素身份验证和单点登录。

身份与访问管理 (IAM) 系统帮助组织机构遵守 HIPAA 、GDPR 和 PCI DSS 等监管要求。它们提供管理用户身份生命周期的功能，包括用户注册、离职和访问权限撤销。此外， IAM 系统还允许组织机构跟踪和审计用户活动，从而有助于防止安全漏洞并确保符合行业法规。

# 八、访问控制模型

系统根据所选模型控制对各种资源的访问。一些常见的访问控制模型包括：

1. 自主访问控制（DAC）
2. 基于角色的访问控制（RBAC)
3. 强制访问控制（MAC）
4. **基于属性的访问控制 (ABAC)** 

### 自主访问控制

许多人在与朋友和同事共享文件或文件夹时已经使用过自主访问控制 (DAC)。使用 DAC 时，资源所有者会明确地添加具有相应权限的用户。

考虑以下示例。您将照片存储在某个在线存储平台上。为了与家人分享所有毕业相关的照片，您逐一添加了他们的帐户，并授予他们访问相应相册的权限。最终，相册权限将显示几个具有查看权限的帐户。

整个过程简单明了，完全由数据所有者掌控。这种方式非常适合与家人或少数公司用户共享数据。然而，当尝试与众多用户共享数据时，尤其是在用户角色随时间发生变化的情况下，共享就会变得棘手。这种情况就引出了基于用户角色的共享方式。

### 基于角色的访问控制

基于角色的访问控制（ RBAC ）采用了一种非常直观的方法。每个用户都拥有一个或多个角色或职能；此外，他们还根据其角色被授权访问不同的资源。

会计人员需要查阅公司账簿，但无需访问研发实验室或相关文件。因此，用户会根据其角色被划分到不同的用户组。系统将根据用户所属的用户组授予相应的权限和访问权限。

根据用户角色进行分类有很多优势。例如，如果用户被赋予新角色，只需将其添加到相应的新用户组即可。此外，如果用户放弃了某个角色，只需将其从原用户组中移除即可。这种方法使维护工作更易于管理，效率更高。

### 强制访问控制

采用强制访问控制 (MAC) 的操作系统会优先考虑安全性，并大幅限制用户权限。此类系统通常用于特定用途或处理高度机密数据。因此，用户无需执行任何超出必要范围的操作。换句话说，用户将无法安装新软件或更改文件权限。

AppArmor 使 Linux 发行版能够支持 MAC。它已经预装在多种 Linux 发行版中，例如 Debian 和 Ubuntu。

SELinux 项目为 Linux 系统提供了一种灵活的 MAC（监控、授权和保护机制）。它是多个 Linux 发行版（例如 Red Hat 和 Fedora）的标准配置。

### 基于属性的访问控制 (ABAC) 

在这种访问控制方式中，对资源的访问权限取决于一组属性，例如用户角色、时间、位置和设备。ABAC 通常用于云环境和 Web 应用程序。通俗地讲，可以想象成一个高度先进的科幻安全系统，它会扫描用户的特定属性。例如，它会检查用户是否来自某个星球、是否携带特定设备，或者是否在特定时间尝试访问某个资源。这就是 ABAC。它就像是未来智能、灵活的安全解决方案。

# 九、单点登录SSO

用户需要访问各种资源来完成日常工作。例如，他们需要访问电子邮件、共享文件和打印机等等。访问这些资源需要用户拥有登录凭据才能成功验证身份。由于用户名和密码的数量众多，这给用户带来了相当大的挑战，尤其是在用户没有在多个系统中重复使用同一密码的情况下。

单点登录 ( SSO ) 解决了这个问题。用户无需记住多个用户名和密码，只需记住一套登录凭据即可。他们可以通过一个系统进行身份验证，从而访问工作所需的其他系统。

传统上，用户需要创建多个密码，例如登录电脑的密码、查看电子邮件的密码以及访问文件共享的密码。记住这么多密码可能很麻烦，尤其是在理想情况下，密码不应该重复使用。更好的方法是要求用户登录一次，即可访问所有需要的服务；这就是单点登录 (SSO) 的作用。

单点登录 (SSO) 允许组织在授予用户访问其工作所需资源的权限之前，先对其进行一次身份验证。我们可以从中获得诸多优势，以下列举几项。

1. 一个强密码：要求用户记住一个强密码比要求他们记住十个不同的强密码更容易接受。
2. 更简单的多因素认证：为每个不同的服务添加多因素认证是一项巨大的任务，需要完成和维护。通过单点登录，多因素认证只需要启用和配置一次。
3. 更简便的支持：密码重置等支持请求变得更加直接，因为它们现在仅限于单个帐户。
4. 效率：用户无需每次访问新服务时都登录。
